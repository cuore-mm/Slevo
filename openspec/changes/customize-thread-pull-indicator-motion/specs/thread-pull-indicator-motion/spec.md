## ADDED Requirements

### Requirement: プル中と更新中でインジケーターモーションを分離する
システムは Thread 画面の下端更新インジケーターにおいて、プル中モードと更新中モードを分離し、各モードで定義された回転挙動を適用しなければならないMUST。

#### Scenario: 更新中は既存どおり右回転表示
- **WHEN** `uiState.isLoading` が `true` の状態でインジケーターが表示される
- **THEN** システムは既存どおり右回転の `ContainedLoadingIndicator` を表示する

#### Scenario: プル中は左回転表示
- **WHEN** `uiState.isLoading` が `false` かつ下端プルで `overscroll > 0` の状態になる
- **THEN** システムはプル量に応じてインジケーターを左回転させる

### Requirement: プル量連動サイズ拡大
システムはプル中インジケーターのサイズをプル量に連動して拡大し、更新閾値で最大サイズに到達させ、閾値超過時は最大サイズを維持しなければならないMUST。

#### Scenario: 閾値未満で段階拡大
- **WHEN** `overscroll` が更新閾値未満で増減する
- **THEN** システムはプル量比率に応じた拡大率を適用し、段階的にサイズを変化させる

#### Scenario: 閾値到達以降は最大サイズを維持
- **WHEN** `overscroll` が更新閾値以上になる
- **THEN** システムはインジケーターサイズを最大値に固定し、それ以上拡大しない

### Requirement: 更新判定ロジック不変
システムはインジケーター表示変更後も、下端更新判定の閾値、arming 条件、haptic 条件、更新発火条件を変更してはならないMUST。

#### Scenario: 既存の更新発火条件を維持
- **WHEN** ユーザーが下端到達後に既存手順でプル更新を実行する
- **THEN** システムは変更前と同じ条件で更新を発火する
