## ADDED Requirements

### Requirement: 下端静止後のみプルアップ更新を発動可能にする
スレッド画面のプルアップ更新は、リストが最下部へ到達した直後の慣性スクロールではなく、最下部でスクロールが停止した状態から開始した上方向ドラッグでのみ発動判定を行わなければならない（MUST）。

#### Scenario: 慣性スクロールの延長では更新が発動しない
- **WHEN** ユーザーが最下部へフリックし、停止前の慣性で下端オーバースクロール領域へ入る
- **THEN** 更新トリガーはアームされず、更新は発動しない

#### Scenario: 最下部静止後のプルアップで更新判定が有効化される
- **WHEN** ユーザーが最下部でスクロール停止した後に上方向へドラッグする
- **THEN** 更新トリガーがアームされ、閾値判定が開始される

### Requirement: 閾値到達時に触覚フィードバックを1回返す
更新判定が有効な状態でプルアップ量が更新閾値を初めて超えたとき、UIは触覚フィードバックを1回だけ発生させなければならない（MUST）。

#### Scenario: 未到達から到達に遷移した瞬間のみ触覚が発生する
- **WHEN** プルアップ量が閾値未満から閾値以上へ遷移する
- **THEN** 触覚フィードバックが1回発生する

#### Scenario: 閾値到達状態を維持しても再通知しない
- **WHEN** 閾値以上の状態でさらにドラッグが継続する
- **THEN** 同一ジェスチャー中に追加の触覚フィードバックは発生しない

### Requirement: 下部インジケーター表示を統一する
スレッド画面の下端更新インジケーターは矢印アイコンを使用せず、`ContainedLoadingIndicator` 系の表示へ統一しなければならない（MUST）。

#### Scenario: プル中は下部に統一インジケーターを表示する
- **WHEN** 更新判定中に下端オーバースクロール量が正である
- **THEN** 画面下部に統一インジケーターが表示される

#### Scenario: 更新実行中も同系統インジケーターを継続表示する
- **WHEN** 更新処理が開始される
- **THEN** 同系統の下部インジケーター表示が維持される
