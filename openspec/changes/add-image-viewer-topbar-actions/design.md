## Context
画像ビューアは現在、表示/非表示を切り替えるトップバーに戻る操作のみを提供している。画像保存と追加アクションの導線が不足しており、閲覧中に操作を開始するまでの手数が多い。

## Goals / Non-Goals
**Goals:**
- 画像ビューアのトップバーに保存ボタンを追加し、現在表示中の画像に対して保存操作を開始できるようにする。
- トップバーにその他ボタンを追加し、`DropdownMenu` で追加アクションを提示できるようにする。
- バー表示状態とメニュー表示状態の整合性を保ち、バーを閉じたときにメニューも閉じる状態管理にする。

**Non-Goals:**
- 画像保存処理のバックエンド仕様変更。
- ドロップダウンメニューに含める全アクションの機能実装詳細の確定。
- 画像ビューア本体のスワイプ/ズーム挙動の変更。

## Decisions
- トップバーの UI は既存 `ImageViewerTopBar` を拡張し、戻るボタンに加えて「保存」「その他」アイコンを追加する。
- その他ボタン押下で `DropdownMenu` を表示し、メニュー開閉状態は `ImageViewerScreen` 側で単一の `Boolean` 状態として管理する。
- バーが非表示へ遷移した場合は、同時に `DropdownMenu` も閉じる制御を入れて、非表示中の浮遊メニューを防ぐ。
- 保存ボタンとメニュー項目選択はイベントコールバックとして画面層から ViewModel/既存処理へ委譲可能な形で接続する。

## Risks / Trade-offs
- [Risk] トップバーの要素追加で表示領域が狭い端末でボタン配置が詰まる可能性がある。  
  → Mitigation: 既存レイアウト幅を維持しつつ、アイコンのみのアクションに限定する。
- [Risk] メニュー開閉とバー表示アニメーションが競合し、意図しない残留表示が起きる可能性がある。  
  → Mitigation: バー表示フラグ変更時にメニュー状態を明示的にクローズする。
