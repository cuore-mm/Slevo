## Context

スレッド画面のレスポップアップは `ReplyPopup` 内で `Column + verticalScroll` を使って投稿一覧を表示している。
現在はスクロール位置を示す UI がなく、長いポップアップで閲覧位置を把握しづらい。
一方でアプリ内では `com.github.nanihadesuka:LazyColumnScrollbar` を既に利用しており、`ScrollbarSelectionMode.Disabled` による非ドラッグ設定の実績がある。

## Goals / Non-Goals

**Goals:**
- レスポップアップ内の投稿一覧にスクロールバーを表示し、現在位置を視覚的に把握できるようにする。
- スクロールバーつまみをドラッグ不可にして、表示専用インジケーターとして扱う。
- 既存のポップアップ操作（返信導線、URL/画像操作、多段ポップアップ）に影響を与えない。

**Non-Goals:**
- スクロール不可時にスクロールバーを非表示化する最適化。
- ポップアップ外のスレッド本体スクロールバー仕様変更。
- ポップアップ表示位置やアニメーション仕様の変更。

## Decisions

- **ポップアップ一覧を `LazyColumn` に置き換え、`LazyColumnScrollbar` でラップする**
  - 非ドラッグ型スクロールバー要件を既存依存で満たせるため、新規ライブラリを追加しない。
  - 代替案として `Column + verticalScroll` に独自スクロールバー描画を重ねる方法もあるが、実装と保守コストが高く採用しない。

- **`ScrollbarSettings` で `selectionMode = ScrollbarSelectionMode.Disabled` を適用する**
  - 要件の「つまみドラッグ不可」を明確に担保する。
  - 代替案としてデフォルト設定を使う方法はドラッグ可になる可能性があり、要件に反するため採用しない。

- **表示対象データと既存イベント伝播は現行仕様を維持する**
  - 投稿要素描画ロジックは維持し、リストコンテナのみ差し替えて影響範囲を限定する。
  - 代替案としてポップアップ描画全体を再設計する方法は、Issue #442 のスコープを超えるため採用しない。

## Risks / Trade-offs

- [Risk] `Column` から `LazyColumn` への変更で項目キーや再構成タイミングが変わり、表示順や区切り線描画が崩れる可能性がある。
  - Mitigation: 投稿番号に基づく安定キーを使用し、既存の区切り線表示条件を同等に再現して確認する。

- [Trade-off] スクロール不可時非表示を今回見送るため、短いポップアップでもスクロールバーが表示される場合がある。
  - Mitigation: 本変更では要件外として明示し、後続チケットで表示条件最適化を切り出して対応する。
