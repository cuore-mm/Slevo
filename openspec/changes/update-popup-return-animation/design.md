## Context
- レスポップアップはAnimatedVisibilityで表示アニメーションを行っている
- 画像ビューアから戻ると既存ポップアップが再描画され、再度アニメーションが走る

## Goals / Non-Goals
- Goals:
  - 画面復帰時に既存ポップアップの表示アニメーションを抑止する
  - 新規ポップアップ追加時のアニメーションは維持する
- Non-Goals:
  - ポップアップの表示ロジック自体の変更
  - 画像ビューア側のトランジション仕様変更

## Decisions
- Decision: ポップアップ表示の入口アニメーションを条件付きで無効化できるフラグを追加し、復帰時のみ無効化する
- Alternatives considered:
  - ポップアップの状態をアニメーション状態ごとViewModelに保持する（実装負荷が高い）
  - 画面復帰時にポップアップを一旦閉じて再表示する（目的と逆で違和感が増える）

## Risks / Trade-offs
- 画面復帰の判定タイミングがずれると、通常表示でもアニメーションが抑止される可能性がある
  - Mitigation: 復帰時の一度だけ無効化し、即座にフラグを戻す

## Migration Plan
- 既存コードにフラグを追加し、復帰時のみに適用する
- 既存のポップアップ追加動作は変更しない

## Open Questions
- 画面復帰判定はLifecycleのON_RESUMEで統一するか、Navigation戻り専用のフックを使うか
