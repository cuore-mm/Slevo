## Context
- スレッド画面には返信番号/返信元/ID を起点としたポップアップ表示が存在する。
- 既存のツリー構造（親子関係）を利用して関連レスをまとめて確認できるようにする。

## Goals / Non-Goals
- Goals:
  - 通常タップで該当レスのツリー全体を素早く確認できる。
  - ツリーがない単独レスでは表示しない。
  - 既存の返信番号/返信元/ID/URL などのタップ挙動は維持する。
- Non-Goals:
  - ツリー構築ロジック自体の変更。
  - ツリー表示のUI刷新や新しい画面遷移の追加。

## Decisions
- Decision: スレッドが保持しているツリー構造（親子関係）を用いて、タップレスの最上位祖先を起点に全子孫を抽出する。
- Decision: ツリー全体ポップアップ内では、通常タップのツリー再表示を無効化し、返信番号/返信元/ID/URL は従来通り動作させる。
- Alternatives considered:
  - 別画面への遷移でツリー表示する案（操作が重くなるため不採用）。
  - ポップアップ内の全タップを無効化する案（既存挙動を壊すため不採用）。

## Risks / Trade-offs
- タップ判定の優先順位が増えるため、既存の押下フィードバックが変化する可能性。
  - Mitigation: 通常タップのみを追加し、リンク/返信/ID のヒット判定は従来順序を維持する。

## Migration Plan
- 既存機能の上書きではなく追加挙動のため、マイグレーションは不要。
