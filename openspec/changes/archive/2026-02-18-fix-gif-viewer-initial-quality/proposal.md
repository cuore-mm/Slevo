## Why

スレッド画面の GIF サムネイルから画像ビューアへ遷移した直後に、初期表示だけ画質が粗くなる不具合が発生している。ユーザーは一度隣の画像へ移動して戻ると高画質になるため、初回表示品質が不安定で体験を損ねている。

## What Changes

- 画像ビューア初期表示時の画像リクエスト生成契約を見直し、初回ページでも表示サイズに整合した品質で描画されるようにする。
- GIF を含む同一 URL のサムネイル表示とビューア表示が連続する場合でも、低解像度キャッシュが初回表示へ固定化されないようにする。
- 既存の共有/保存向け再利用メタデータ登録（`diskCacheKey` 等）と進捗表示契約を維持したまま、表示品質の回帰を防ぐ。
- 画像ビューア画質に関する再発防止のため、初回表示品質の検証観点をタスク化する。

## Capabilities

### New Capabilities
- なし

### Modified Capabilities
- `image-viewer`: 画像ビューアの初回表示品質契約を追加し、GIF サムネイル起点でも初回から高解像度表示される要件を明確化する。

## Impact

- 影響範囲は画像ビューアのページ描画処理（`ImageViewerPager`）と、必要に応じて同画面のリクエスト生成関連処理。
- 既存の shared transition、画像共有/保存前処理、読み込み進捗表示との整合確認が必要。
- API 仕様変更や外部依存追加は想定しない。
