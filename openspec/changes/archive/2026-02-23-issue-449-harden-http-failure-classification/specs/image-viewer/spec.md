## ADDED Requirements

### Requirement: 404/410 専用表示は HTTP 応答コード由来で判定する
システムは画像読み込み失敗を 404/410 専用表示へ分類する際、例外メッセージ文字列の数値一致ではなく、HTTP 例外が保持する応答ステータスコードを根拠に判定しなければならないMUST。

#### Scenario: HTTP 404 応答は 404 専用表示になる
- **WHEN** 画像読み込みが HTTP 404 応答で失敗する
- **THEN** システムは `HTTP_404` として分類し、404 専用表示を適用する

#### Scenario: HTTP 410 応答は 410 専用表示になる
- **WHEN** 画像読み込みが HTTP 410 応答で失敗する
- **THEN** システムは `HTTP_410` として分類し、410 専用表示を適用する

### Requirement: HTTP ステータス不明な失敗は UNKNOWN として扱う
システムは HTTP 応答コードを取得できない画像読み込み失敗を 404/410 と推測分類してはならないMUST。分類不能な失敗は `UNKNOWN` として扱い、既存の再試行導線を維持しなければならないMUST。

#### Scenario: メッセージに 404 数字を含む非 HTTP 失敗は UNKNOWN のまま
- **WHEN** 非 HTTP 失敗の例外メッセージに `404` または `410` の数字が含まれる
- **THEN** システムは 404/410 専用分類を行わず `UNKNOWN` として扱う

#### Scenario: HTTP 500 応答は専用表示に分類されない
- **WHEN** 画像読み込みが HTTP 500 応答で失敗する
- **THEN** システムは `UNKNOWN` として扱い、404/410 専用表示を適用しない
