## Why

issue #449 の 404/410 表示導入では、失敗種別判定に例外メッセージ内の数値抽出を使っているため、HTTP 応答由来でない失敗でも 404/410 と誤分類される余地があります。誤分類時に再試行導線が抑止されると復旧可能な失敗を回復できないため、失敗分類を HTTP ステータスに基づく厳密判定へ変更します。

## What Changes

- 画像読み込み失敗の 404/410 判定を「例外メッセージ文字列」ではなく「型付き HTTP 例外のステータスコード」から行う。
- HTTP ステータスを取得できない失敗は `UNKNOWN` として扱い、既存の再試行導線を維持する。
- 404/410 専用表示は厳密に HTTP 404/410 応答時のみ適用する。

## Capabilities

### New Capabilities
- なし

### Modified Capabilities
- `image-viewer`: 画像読み込み失敗分類の判定根拠を厳密化し、404/410 専用表示の適用条件を HTTP 応答コード起点へ変更する。

## Impact

- 対象コード: `ui/util/ImageLoadFailureType.kt` を中心に、`toImageLoadFailureType()` 呼び出し元（投稿サムネイル/ビューア本体/下部サムネイル）。
- 仕様影響: 404/410 表示の誤発火を抑止し、非 HTTP 失敗の再試行可能性を維持する。
- テスト観点: HTTP 404/410、非 HTTP 失敗（タイムアウト/デコード失敗等）、HTTP 500 の分類結果確認。
