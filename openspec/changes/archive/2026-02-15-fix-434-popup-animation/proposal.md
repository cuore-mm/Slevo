## Why

スレッド画面でレスポップアップを表示した状態で画像ビューアへ遷移し、戻ってきた際にポップアップの表示アニメーションが再実行される不具合が発生している。復帰時は既存ポップアップの継続表示として扱うべきであり、不要なアニメーションは視認性と操作の連続性を損なうため、仕様として明確化して修正する必要がある。

## What Changes

- スレッド画面復帰時のレスポップアップ表示挙動を定義し、既に表示中のポップアップでは再表示アニメーションを発生させない。
- 新規に追加されたポップアップのみ表示アニメーションを行う既存挙動を維持する。
- ポップアップの閉鎖アニメーションや階層表示、タップ操作など既存の挙動を維持したまま、復帰時の初期化境界のみを見直す。

## Capabilities

### New Capabilities
- なし

### Modified Capabilities
- `thread-tree-popup`: 画面復帰時の既存ポップアップ再表示で enter アニメーションを抑止する要件を追加する。

## Impact

- 影響範囲はスレッド画面のレスポップアップ描画ロジック（Compose の表示状態同期）
- 画像ビューアからの戻り導線を含む画面復帰時の UI 表示品質
- API や外部依存関係の変更はない
