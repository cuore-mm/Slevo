## Context

画像ビューアは没入表示を優先しており、背景・バー・ツールチップの配色を個別に固定値で扱っている箇所がある。現状はライト/ダークモードとの連動要件が仕様化されていないため、モード切替時に前景色とのコントラストが一貫しない。

今回の変更は `image-viewer` capability の表示仕様を拡張し、画面背景、バー背景、ツールチップ背景、文字/アイコン前景色をモードで統一的に切り替えることを目的とする。

## Goals / Non-Goals

**Goals:**
- 画像ビューアの主要 UI 要素をライト/ダークモードで明確に切り替え、可読性を担保する。
- 背景色とバー/ツールチップの半透明背景を同一ルールで定義し、画面内の配色一貫性を確立する。
- 既存の操作契約（スワイプ、保存、メニュー、ツールチップ表示）を維持したまま配色のみ変更する。

**Non-Goals:**
- テーマシステム全体の再設計や他画面への配色展開。
- 新規アニメーションや操作フローの追加。
- Material3 の既定カラーパレット全面置換。

## Decisions

- **配色判定の基準を表示モードに統一する**
  - 画像ビューア内の背景・バー・ツールチップ・前景色は、同一の「ライトかダークか」判定結果を参照して決定する。
  - 代替案として要素ごとの独立判定もあるが、状態不整合（背景だけ切替済み等）を避けるため採用しない。

- **背景とバー/ツールチップを分離した固定ルールで定義する**
  - 背景はライト=白、ダーク=黒の不透明色を使う。
  - バーとツールチップはライト=半透明白、ダーク=半透明黒を使い、コンテンツ視認性と情報重なりのバランスを取る。
  - 代替案の動的トーン計算は柔軟だが、今回の受け入れ条件は明示色指定のため採用しない。

- **前景色は背景コントラスト優先で反転させる**
  - 背景が明色のときは暗色文字/アイコン、背景が暗色のときは明色文字/アイコンを適用する。
  - これによりトップバーアイコン、ツールチップ文字、必要なラベルの視認性を一貫させる。

## Risks / Trade-offs

- [Risk] 半透明バー色が端末や壁紙の見え方差で想定より薄く見える可能性 → Mitigation: 既存 UI 上で実機確認し、必要ならアルファ値のみ微調整可能な実装にする。
- [Risk] 既存のツールチップスタイルと衝突し、意図しない色が適用される可能性 → Mitigation: 画像ビューア内ツールチップに適用範囲を限定し、他画面のツールチップ仕様に影響させない。
- [Trade-off] 明示的な白/黒ベース指定によりブランド色の自由度は下がる → 受け入れ条件準拠を優先し、将来的なテーマ拡張時に再評価する。
