## ADDED Requirements

### Requirement: 一括保存メニューの複数対象アイコン表示
システムは `ImageMenu` の BottomSheet に表示する「レス内の画像をすべて保存」項目に、`download` をベースとして右下に `filter_none` バッジを重ねた合成アイコンを表示しなければならないMUST。合成描画はベースアイコンとバッジアイコンを引数で指定できる共通実装で行わなければならないMUST。重なり部分ではバッジ背面をマスクし、ベースアイコンが透けて見えない表示を維持しなければならないMUST。この要件は単体保存との視覚的識別を目的とし、既存の一括保存実行条件および保存動作契約を変更してはならないMUST。

#### Scenario: すべて保存項目で合成アイコンを表示する
- **WHEN** ユーザーが複数画像を含むレスの画像メニューを開く
- **THEN** 「レス内の画像をすべて保存」項目に download + 重なりバッジの合成アイコンが表示される

#### Scenario: 重なり部分でベースアイコンが透けない
- **WHEN** ユーザーが「レス内の画像をすべて保存」の合成アイコンを表示する
- **THEN** バッジの重なり領域でベースアイコンは視認されず、バッジが明確に判別できる

#### Scenario: 単体保存項目の表示契約は維持する
- **WHEN** ユーザーが同じ画像メニューで「この画像を保存」を確認する
- **THEN** 単体保存項目は既存契約どおりの表示を維持し、追加した複数対象バッジは表示されない

### Requirement: コピー系メニューの統一バッジ表示
システムは `ImageMenu` の BottomSheet に表示する「この画像をコピー」と「画像URLをコピー」の両項目で、`content_copy` をベースとして右下に `filter_none` バッジを重ねた同一の合成アイコンを表示しなければならないMUST。コピー系も保存系と同一の共通合成実装を利用しなければならないMUST。両項目の挙動契約は既存どおり維持しなければならないMUST。

#### Scenario: 画像本体コピーと画像URLコピーで同じバッジ付きアイコンを表示する
- **WHEN** ユーザーが画像メニューを開く
- **THEN** 「この画像をコピー」と「画像URLをコピー」の両項目に同一のバッジ付きコピーアイコンが表示される
