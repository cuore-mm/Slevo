## ADDED Requirements

### Requirement: 共通アイコンボタン部品の提供
システムは押下時の視覚フィードバックと長押しツールチップ表示を備えた `FeedbackTooltipIconButton` を共通 UI コンポーネントとして提供しなければならないMUST。

#### Scenario: 共通部品で押下フィードバックを表示する
- **WHEN** ユーザーが対象画面のアイコンボタンを押下する
- **THEN** システムは押下中のみ縮小表示し、押下解除で通常サイズに復帰する

#### Scenario: 共通部品で長押しツールチップを表示する
- **WHEN** ユーザーが対象画面のアイコンボタンを長押しする
- **THEN** システムは当該ボタンの機能を示すツールチップを表示する

### Requirement: 板/スレッド/検索/設定トップバーへの統一適用
システムは板画面とスレッド画面のボトムバー内ボタン、検索バー内ボタン、設定画面トップバー内ボタンを `FeedbackTooltipIconButton` ベースへ統一しなければならないMUST。置換後も既存のクリック時アクション契約を変更してはならないMUST。

#### Scenario: 板画面とスレッド画面のボトムバーが統一部品で動作する
- **WHEN** ユーザーが板画面またはスレッド画面のボトムバー内ボタンを操作する
- **THEN** システムは `FeedbackTooltipIconButton` の共通挙動で表示しつつ、既存と同じアクションを実行する

#### Scenario: 検索バー内ボタンが統一部品で動作する
- **WHEN** ユーザーが検索バー内の戻る、クリア、音声入力ボタンを操作する
- **THEN** システムは `FeedbackTooltipIconButton` の共通挙動で表示しつつ、既存と同じ検索操作契約を維持する

#### Scenario: 設定画面トップバー内ボタンが統一部品で動作する
- **WHEN** ユーザーが設定画面トップバーの戻るまたはメニューボタンを操作する
- **THEN** システムは `FeedbackTooltipIconButton` の共通挙動で表示しつつ、既存と同じ遷移/メニュー表示契約を維持する

### Requirement: ジェスチャー設定メニューのアンカーオーバーレイ化
システムはジェスチャー設定画面のトップバーから開くメニューを `DropdownMenu` ではなく `AnchoredOverlayMenu` で表示しなければならないMUST。メニューはアンカーボタンを基準に重ね表示し、dismiss 契約を維持しなければならないMUST。

#### Scenario: メニューボタン押下でアンカー重ね表示される
- **WHEN** ユーザーがジェスチャー設定画面トップバーのメニューボタンを押下する
- **THEN** システムは `AnchoredOverlayMenu` をアンカーボタン基準で重ね表示する

#### Scenario: メニュー外タップや戻る操作で閉じる
- **WHEN** ユーザーがメニュー外をタップするか戻る操作を行う
- **THEN** システムは `AnchoredOverlayMenu` を dismiss し、既存と同じ終了契約を維持する
